# Security Policy

## Supported Versions

Seedling is currently maintained from the default branch only.

## Reporting A Vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Send a private report through GitHub's private advisory flow in the repository Security tab using `Security` -> `Report a vulnerability`, and include:
- a clear description of the issue
- affected files or flows
- reproduction steps or proof of concept
- impact assessment

Initial triage should happen within 5 business days. Please allow reasonable time for remediation before public disclosure.

## Scope Notes

Seedling is a privacy-focused mobile app. Reports about:
- accidental secret exposure
- encryption or storage weaknesses
- sync/auth flaws
- widget or export privacy leaks
- unsafe release/signing practices

are especially useful.
